# MEC-0004 — Action-first interactive learning

## Pattern

Give only enough explanation to make an action meaningful, then let the learner manipulate or predict through a representation. Feedback refers to the current state and supports revision before the next conceptual step.

This is a learning-interaction pattern rather than a complete game mechanic. It should combine with a concept-native source of decisions and challenge.

## Core loop

```text
brief orientation → learner action/prediction
→ visible consequence → state-specific feedback or hint
→ revision/explanation → next conceptual checkpoint
```

## Meaningful decisions

Meaning must come from the paired concept mechanic: organizing, constructing, sequencing, estimating, comparing, or testing. Immediate feedback alone does not create agency.

## Challenge structure

- **State representation:** manipulable mathematical or scientific model.
- **Challenge:** progressively coordinated representations or applied constraints.
- **Failure information:** feedback points to the current state and relationship.
- **Recovery:** contextual hint, guided revision, or prerequisite checkpoint.
- **Depth:** increasing reasoning demand rather than speed or cosmetic variation.

## Potential learning affordances

- Reduce separation between explanation and use.
- Tie feedback to visible evidence.
- Coordinate concrete, visual, and symbolic representations.
- Revisit a prerequisite at a meaningful checkpoint.

## Poor fits and risks

- An interactive diagram with no meaningful decision is still passive.
- Immediate correctness feedback can collapse into trial-and-error clicking.
- Platform engagement systems such as XP or streaks are not learning evidence.
- Copying a polished screen can hide the absence of a concept-native loop.

## Adaptation questions

- What action makes the target relationship visible?
- What specific current-state evidence should feedback reference?
- Which support fades by the independent scene?
- What prerequisite checkpoint is genuinely necessary?

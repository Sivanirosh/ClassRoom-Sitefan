# PRE-0005 — Opus Magnum

## Why this record exists

Opus Magnum provides a precedent for accepting several correct constructions before inviting optional refinement rather than enforcing one hidden route.

## Source-backed loop

- **Player verbs:** design and build machines from available components to perform requested processes.
- **Completion:** a machine must carry out the required process.
- **Refinement:** working solutions can be compared for simplicity, speed, and compactness.
- **Inspection/sharing:** animated output can expose machine operation.
- **Unknown detail:** the located first-party pages do not fully document components, command semantics, collisions, execution ordering, or editor recovery.

## Transferable observations

A learning mechanic can accept multiple correct constructions and then expose independent optional goals. This can reveal strategy without treating the most compact or fastest solution as the only correct one.

## Unknowns and cautions

- Do not add detailed simulation claims absent a first-party manual.
- Optimization metrics can distract from conceptual evidence and should remain optional.
- Open construction needs bounded content and inspectable validity rules.
